Each cadence task scores against its own timing rule, set the outcome from its dropdown. Bonus points come from logging real milestones and customer actions as they happen, no cap. Grading is now percentage of that quarter's own max delivery score, since Q1 has far more scorable tasks (max 45) than Q2–Q4 (max 12, 9, 12) — a flat point threshold would've made the later quarters nearly impossible to clear on delivery alone. Bonus points count as pure upside on top, so a quarter can run past 100%. "Formal renewal proposal delivered" (Month 11) didn't come with a specified point value, it's defaulted to 3/0 to match the other one-time deliverables, flag if that's wrong. Data syncs to GitHub, shared by anyone with a token for this repo.
